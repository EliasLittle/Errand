[Blog post](https://onebigfluke.com/2022/11/the-case-for-dynamic-functional.html) on why Dynamic typing and functional programming both make it easier to reason about your code.

This is why dynamic typing should be allowed (much easier to reason about code), but we still want the safety of statically typed (and checked) code. 

Thus allow both via compiler options. 
Workflow would be:
dev  --> dynamic 
prod --> static
